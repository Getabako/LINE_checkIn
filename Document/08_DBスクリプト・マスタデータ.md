# 08. DB スクリプト・マスタデータ

Firestore は RDB と異なり **スキーマ定義を流す必要はない**（コレクション／ドキュメントは書き込み時に自動生成）。  
ここでは初期マスタデータの **JSON 仕様** と **投入スクリプト** を掲載する。

- スクリプト: [`./scripts/seed.mjs`](./scripts/seed.mjs)
- マスタデータ:
  - [`./master-data/memberTypes.json`](./master-data/memberTypes.json)
  - [`./master-data/coupons.sample.json`](./master-data/coupons.sample.json)
  - [`./master-data/announcements.sample.json`](./master-data/announcements.sample.json)

---

## 1. 実行手順

```bash
# 事前: Firebase サービスアカウント JSON を用意（Base64 化前の JSON）
export GOOGLE_APPLICATION_CREDENTIALS="/abs/path/firebase-service-account.json"

# 会員種別マスタの初期投入
node Document/scripts/seed.mjs memberTypes

# クーポンサンプル
node Document/scripts/seed.mjs coupons

# お知らせサンプル
node Document/scripts/seed.mjs announcements
```

`FIREBASE_SERVICE_ACCOUNT_KEY`（Base64）をそのまま使う場合:

```bash
export FIREBASE_SERVICE_ACCOUNT_KEY="<base64>"
node Document/scripts/seed.mjs memberTypes
```

## 5. バックアップ / リストア

### 5.1 手動エクスポート

```bash
# gcloud CLI が必要
gcloud firestore export gs://<bucket>/backup/$(date +%Y%m%d)
```

### 5.2 定期バックアップ

Firebase Console の **Firestore → インポート / エクスポート** でスケジュール設定（日次推奨）。

### 5.3 リストア

```bash
gcloud firestore import gs://<bucket>/backup/20260101
```

> ⚠️ 既存データは上書きされる。`--collection-ids` で対象を限定可能。

## 6. データ確認 SQL 風クエリ（Firestore Console から）

Firestore Console の **データ**タブで以下のフィルタを指定する。

| 目的 | コレクション | フィルタ |
|---|---|---|
| 今日の予約 | `checkins` | `date == 2026-05-01` |
| クーポン利用履歴 | `couponRedemptions` | `couponId == <id>` |
| アクティブ会員 | `userMemberships` | `isActive == true` |
| 未決済の予約 | `checkins` | `status == PENDING` |
