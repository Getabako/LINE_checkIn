# 05. 開発環境構築手順

## 1. 前提ツール

| ツール | バージョン | 用途 |
|---|---|---|
| Node.js | **20.x**（LTS） | ランタイム（`engines.node: "20.x"` で固定） |
| npm | 10.x 以上 | パッケージ管理 |
| Git | 任意 | |
| Vercel CLI | 最新（`npm i -g vercel@latest`） | `vercel dev`, `vercel env pull` 等 |
| Firebase CLI | 任意 | Firestore 操作 |

> macOS では `brew install node@20` 推奨。Windows は nvm-windows を使用。

---

## 2. リポジトリ取得

```bash
git clone <repo-url>
cd LINEMiniApp/gym-checkin
```

## 3. パッケージインストール

`package-lock.json` をそのまま使うため `npm ci` を推奨。

```bash
npm ci
```

> ⚠️ `npm install` でも動くが lock が更新される。本番と同じ依存を使うなら `ci`。

---

## 4. 環境変数設定（.env）

ルート直下に `.env`（git 管理外）を作成する。値の取得先は [07. 動作環境構築手順](./07_動作環境構築手順.md) を参照。

```env
# LIFF
VITE_LIFF_ID=2008793930-iDPTCREA

# Firebase (Base64 JSON)
FIREBASE_SERVICE_ACCOUNT_KEY=<base64>

# Stripe
STRIPE_SECRET_KEY=sk_test_xxx
STRIPE_WEBHOOK_SECRET=whsec_xxx

# RemoteLock
REMOTELOCK_CLIENT_ID=xxx
REMOTELOCK_CLIENT_SECRET=xxx
REMOTELOCK_DEVICE_ID_ENTRANCE=xxx
REMOTELOCK_DEVICE_ID_GYM=xxx
REMOTELOCK_DEVICE_ID_TRAINING=xxx
REMOTELOCK_DEVICE_ID_YABASE=xxx

# 管理者
ADMIN_LINE_USER_IDS=Uxxxxxxxx,Uyyyyyyyy

# 開発用
NEXT_PUBLIC_BASE_URL=http://localhost:5173
SKIP_PAYMENT=false
LOG_LEVEL=debug
```

`vercel env pull .env` で Vercel プロジェクトから取得することも可能。

---

## 5. 開発サーバー起動

### 5.1 Vite（フロントのみ）

```bash
npm run dev
# http://localhost:5173
```

- `VITE_LIFF_ID` が未設定のときは **LIFF モック**で動作（`U_dev_user_12345`）。
- API は `VITE_API_URL` が未設定なら `/api` にプロキシされる。

### 5.2 Vercel Dev（/api 付き）

```bash
vercel link      # 初回のみ
vercel dev       # フロント + /api を同時起動
```

- Vercel CLI が `.env` を読み込み、Serverless Functions もローカルで動作させる。
- Stripe Webhook をローカルで受けたい場合は `stripe listen --forward-to localhost:3000/api/payments/stripe-webhook`。

## 7. デバッグ Tips

- `DebugPanel`（画面下の小さなゲージ）で **支払いスキップ**・**RemoteLock スキップ**を切替可能
- LIFF モック時は LINE にアクセスせず、`mock-access-token-for-development` をサーバー側で許容
- `LOG_LEVEL=debug` を設定すると、サーバーロガーの `debug` ログも出る
- ブラウザ DevTools の Console に `[OPERATION]` で操作ログが出る

## 8. ありがちな詰まりどころ

| 症状 | 原因 | 対処 |
|---|---|---|
| `FIREBASE_SERVICE_ACCOUNT_KEY is not set` | `.env` 未設定 or vercel dev 未使用 | `.env` を置く／`vercel env pull` |
| LIFF 画面で白画面 | `VITE_LIFF_ID` 未設定で本番モード | `.env` に `VITE_LIFF_ID=` を設定 |
| `npm ci` で Prisma 系エラー | 他 PJ の依存が混入 | このフォルダ内で実行しているか確認 |
| Stripe Webhook 404 | `vercel dev` ではなく `npm run dev` を使っている | `vercel dev` に切り替え |
| Firestore 権限エラー | サービスアカウントのロール不足 | `Cloud Datastore User` を付与 |
