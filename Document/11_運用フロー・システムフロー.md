# 11. 運用フロー・システムフロー

フローは **Mermaid** 記法で記述しています。GitHub / 対応エディタ上でそのまま図として表示されます。

> 💡 Mermaid 8.8 系など古いバージョンでも描画できるよう、`graph` 記法・ダブルクォートでのラベル保護で統一しています。

graph TD

  C --> D["施設選択"]
  D --> E{"予約タイプ"}
  E -->|"単日"| F["単日予約入力"]
  E -->|"複数日"| G["複数日予約入力"]
  E -->|"定期"| H["毎週 / 隔週を指定"]
  F --> I["確認・クーポン入力"]
  G --> I
  H --> I
  I --> J["Stripe 決済"]

## 3. システムフロー — 予約〜決済〜PIN 発行（Stripe モード）

```mermaid
sequenceDiagram
  autonumber
  participant U as "利用者 (LIFF)"
  participant F as "フロント (React)"
  participant API as "/api"
  participant FS as "Firestore"
  participant ST as "Stripe"
  participant RL as "RemoteLock"

  U->>F: 拠点・施設・日時選択
  F->>API: POST /api/prices/calculate
  API-->>F: 金額
  F->>API: POST /api/coupons/validate
  API-->>F: 割引額
  U->>F: カードで支払う
  F->>API: POST /api/payments/create-checkout
  API->>FS: checkins を PENDING で作成 / groupId 設定
  API->>ST: Checkout Session 作成
  ST-->>API: checkoutUrl
  API-->>F: checkoutUrl
  F->>ST: リダイレクト
  U->>ST: カード / PayPay 入力
  ST-->>U: 決済成功
  ST->>API: POST /api/payments/stripe-webhook
  API->>FS: checkin 取得 (PENDING)
  API->>RL: POST /bookings (玄関 → 施設)
  RL-->>API: pin
  API->>FS: status=PAID, pinCode 更新
  API-->>ST: 200 received
  F->>API: GET /api/checkins/:id
  API-->>F: pinCode
  F-->>U: 完了画面で PIN 表示
```

## 4. システムフロー — 決済スキップ（管理者予約／デバッグ）

```mermaid
sequenceDiagram
  autonumber
  participant ADMIN as "管理者 / 開発者"
  participant API as "/api"
  participant FS as "Firestore"
  participant RL as "RemoteLock"

  ADMIN->>API: POST /api/payments/create-checkout (skipPayment=true)
  API->>FS: checkins を PENDING で作成
  API->>RL: POST /bookings (各日)
  RL-->>API: pin (1本目を全日で共有)
  API->>FS: status=PAID, pinCode 更新
  API-->>ADMIN: mode=skip, checkinIds, groupId
```

## 5. システムフロー — キャンセル

```mermaid
sequenceDiagram
  autonumber
  participant U as "利用者"
  participant F as "フロント"
  participant API as "/api/checkins/:id"
  participant FS as "Firestore"
  participant RL as "RemoteLock"

  U->>F: キャンセルする
  F->>API: DELETE /api/checkins/:id
  API->>FS: 対象取得・時間前チェック
  alt 利用開始1時間以上前
    API->>RL: PUT /bookings/:id/deactivate
    API->>FS: status=CANCELLED
    API-->>F: 200 OK
  else 1時間を切っている
    API-->>F: 409 キャンセル不可
  end
```
