# 03. DB 設計書（Firestore）

データベースは **Firebase Firestore** を使用する。Prisma/Neon から移行済み。

## 0. 全体

- プロジェクト: Firebase Console で作成した `gym-checkin-xxxx`
- モード: **Native mode**
- リージョン: `asia-northeast1`（東京）推奨
- 認証: サーバーサイドは `firebase-admin` の **サービスアカウント** を利用（環境変数 `FIREBASE_SERVICE_ACCOUNT_KEY`、Base64 JSON）

### コレクション一覧

- ユーザー別: `where userId == ?`（メモリで createdAt desc）

> ℹ️ 複合インデックス回避のため、2 条件目以降はメモリでフィルタしている。

## セキュリティルール（推奨）

本アプリは **サーバー経由でのみ Firestore にアクセスする** 設計のため、クライアントから直接アクセスする経路は用意していません。Firestore ルールは **全拒否** が推奨です。

```
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    match /{document=**} {
      allow read, write: if false;
    }
  }
}
```

## インデックス

現状は **複合インデックス不要**（単一フィールドクエリ＋メモリ処理で統一）。
Firestore コンソールから複合インデックスが要求された場合は、該当クエリをメモリ処理に書き換えて回避する方針。
