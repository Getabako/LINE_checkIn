# 06. ビルド・デプロイ手順

## 1. ビルドパイプライン

```
npm run build
  ├─ tsc -b       # 型チェック（Project References）
  └─ vite build   # フロントを dist/ に出力
```

Vercel 上では `vercel.json` の `buildCommand: "npm run build"` が実行される。
`/api/**` のファイルは Vercel が自動的に **Serverless Functions** としてビルドする。

### 1.1 ローカル確認

```bash
npm run build
npm run preview   # dist/ を 4173 番で起動
```

---

## 2. Vercel プロジェクト初期設定

### 2.1 GitHub 連携

1. GitHub リポジトリにプッシュ
2. Vercel ダッシュボード → **Add New → Project**
3. リポジトリをインポート
4. **Root Directory**: `LINEMiniApp/gym-checkin`
5. **Framework Preset**: `Vite`（自動検出）
6. ビルド／出力は `vercel.json` の指定を優先

### 2.2 初回 CLI リンク

```bash
cd LINEMiniApp/gym-checkin
npx vercel link
# > プロジェクトを選択
```

### 2.3 環境変数を Vercel に投入

[07. 動作環境構築手順](./07_動作環境構築手順.md) の一覧にある **すべての環境変数** を Production と Preview の両方に登録する。

```bash
# CLI から
npx vercel env add VITE_LIFF_ID production
npx vercel env add FIREBASE_SERVICE_ACCOUNT_KEY production
# ...（全項目）
```

GUI の場合: Project → **Settings → Environment Variables**。

> ⚠️ `FIREBASE_SERVICE_ACCOUNT_KEY` は **Base64 エンコードした JSON 文字列** を登録する。

```bash
# macOS
cat firebase-service-account.json | base64 | tr -d '\n' | pbcopy
```

## 3. デプロイ方法

### 3.1 Production デプロイ

| 方法 | 手順 |
|---|---|
| GitHub プッシュ | `main` ブランチに push するだけで自動デプロイ（推奨） |
| CLI | `npx vercel --prod` |

### 3.2 Preview デプロイ

- PR を作成すると自動で Preview URL が発行される
- CLI: `npx vercel` （Preview）

### 3.3 Rollback（ロールバック）

Vercel ダッシュボード → **Deployments** → 任意の過去デプロイ → **Promote to Production**。

## 4. デプロイ前チェックリスト

- [ ] `npm ci && npm run build` がローカルで成功する
- [ ] `package-lock.json` がコミットされている
- [ ] `node` バージョンが 20.x 固定（`package.json engines`）
- [ ] `.env` は commit されていない（`.gitignore` 確認）
- [ ] Firestore/Stripe/RemoteLock の環境変数が **Production** に設定済
- [ ] Stripe Webhook の URL を本番 URL に更新している
- [ ] LIFF エンドポイント URL を本番 URL に変更している
- [ ] `ADMIN_LINE_USER_IDS` を本番運営者の LINE userId に更新している

---

## 5. デプロイ後確認

1. 本番 URL を LINE Developers Console で LIFF の **Endpoint URL** に設定
2. LINE 公式アカウントのリッチメニューから LIFF URL を起動
3. LIFF 画面が白画面にならず「拠点選択」が表示されることを確認
4. テスト予約 → Stripe テストカード（`4242 4242 4242 4242`）→ 完了画面で PIN が表示されることを確認
5. RemoteLock 管理画面側で `bookings` が作成されているか確認
6. Vercel **Logs** で `[OPERATION]` ログが流れているか確認

## 6. CI/CD の補足

- 本プロジェクトは Vercel の GitHub Integration のみで CI を構成している
- 追加で GitHub Actions を導入する場合の例:

```yaml
# .github/workflows/check.yml
name: lint-and-build
on:
  pull_request:
jobs:
  build:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with:
          node-version: '20'
          cache: 'npm'
          cache-dependency-path: LINEMiniApp/gym-checkin/package-lock.json
      - working-directory: LINEMiniApp/gym-checkin
        run: |
          npm ci
          npm run lint
          npm run build
```

## 7. デプロイ時のトラブルシュート

| 症状 | 対処 |
|---|---|
| `Function exceeds 12 limit` | 新規 API を作らず、`/api/admin` 等の既存関数に `action` を足す |
| `FIREBASE_SERVICE_ACCOUNT_KEY is not set` | Production / Preview の両方に設定されているか確認 |
| Stripe Webhook が 401 | Webhook シークレットが本番用 `whsec_*` か確認 |
| LIFF が真っ白 | Endpoint URL の末尾に `/` を含めない／`VITE_LIFF_ID` を設定 |
| `NEXT_PUBLIC_BASE_URL` の不一致 | 本番では `https://<production-domain>` に設定 |
