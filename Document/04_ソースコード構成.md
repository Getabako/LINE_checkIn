## 2. 主要モジュール解説

### 2.1 `src/App.tsx`

- LIFF 初期化（`initializeLiff`）→ `userApi.getMe()` で Firestore に自動登録
- `BrowserRouter` の下に全ルートを定義
- `DebugPanel` を常時マウント（本番でも長押しで開く開発用）
- 起動時に `logger.op('app.boot')` を出力

### 2.2 `src/lib/liff.ts`

- 開発モード `import.meta.env.DEV && !VITE_LIFF_ID` のときはモックプロファイルで継続
- 本番で `VITE_LIFF_ID` 未設定のときはエラー
- `getAccessToken()` が Bearer として送信される

### 2.3 `src/lib/api.ts`

- `fetch` ベースの薄いクライアント
- ドメイン別に `userApi` / `checkinApi` / `priceApi` / `couponApi` / `memberTypeApi` / `membershipApi` / `reviewApi` / `eventApi` / `schoolApi` / `announcementApi` / `adminApi` を公開
- 型 `LocationId` / `FacilityType` / `CheckinStatus` / `Checkin` / `Coupon` / `MemberType` / ... を共通化

### 2.4 `src/stores/checkinStore.ts`

- Zustand の単一 store（永続化なし）
- 単日・複数日・定期予約の状態をまとめて保持
- `setMultiDateMode(false)` で定期予約情報もクリア

### 2.5 `src/stores/debugStore.ts`

- `paymentEnabled` / `remoteLockEnabled` の 2 フラグを localStorage に保存
- Payment／RemoteLock をスキップしてテストできるトグル

### 2.6 `server-lib/firebase.ts`

- 環境変数 `FIREBASE_SERVICE_ACCOUNT_KEY`（Base64 JSON）を復号して `firebase-admin` を初期化
- `COLLECTIONS` 定数で 12 コレクション名を集約

### 2.7 `server-lib/auth.ts`

- `Authorization: Bearer <token>` を取り出し、`https://api.line.me/v2/profile` で検証
- 開発環境では `mock-access-token-for-development` を許容

### 2.8 `server-lib/remotelock.ts`

- OAuth2 Client Credentials → 5 分マージンで accessToken をキャッシュ
- `getDeviceIds(location, facilityType)` で対象デバイスを決定
  - ASP: `ENTRANCE` + (`GYM` or `TRAINING`)
  - YABASE: `YABASE` 1 台のみ
- `createBooking({ pin })` でグループ予約時に PIN を引き継げる
- `cancelBooking(checkinId, deviceCount)` でキャンセル

### 2.9 `server-lib/pdf.ts`

- `pdf-lib` + `@pdf-lib/fontkit` で日本語 TTF を埋め込み領収書 PDF を生成
- `/api/checkins/[id]?format=receipt` / `/api/admin?action=receipt` から呼び出す

### 2.10 ログ (`src/lib/logger.ts` / `server-lib/logger.ts`)

- フロント: コンソール向け読みやすい整形（`[LEVEL] HH:mm:ss.SSS [scope] ...`）
- サーバー: 1 行 JSON（Vercel ログの検索に最適）
- 両者とも `createLogger(scope)` → `{ debug, info, warn, error, op }`
- 操作ログは `op(action, payload)` を必ず使う

### 2.11 `src/components/common/DebugPanel.tsx`

- `debugStore` のフラグを UI で切り替える開発支援パネル
- 本番でも HTML 内に存在するが、トリガーを長押しにしているため通常利用者には見えない

## 3. ログを差し込む方針

### 3.1 フロント

- 画面の `useEffect` に `log.op('<page>.view')` を入れる（必要に応じ拡張）
- ボタン操作は `onClick` 内で `log.op('<page>.<action>', { ...payload })`
- エラー境界・catch 句に `log.error`

### 3.2 サーバー

- 各ハンドラの冒頭で `log.debug('request.in', { method, query })`
- 認可失敗: `log.warn('unauthorized')`
- 成功の監査ログ: `log.op('<domain>.<action>', { lineUserId, ... })`
- 例外: `log.error('handler.fail', { message: String(error) })`

## 4. ビルド／実行コマンド

