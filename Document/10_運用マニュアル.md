# 10. 運用マニュアル（管理者・運営者向け）

本書は運営スタッフ・システム管理者が **日常運用** で行う作業をまとめたものです。

---

## 1. 管理画面へのアクセス

- URL: `https://<production-domain>/admin`
- 認可: `ADMIN_LINE_USER_IDS` に登録された LINE アカウントのみ利用可能
- 画面上部タブで **カレンダー / イベント / スクール / 会員 / クーポン / お知らせ / 売上** を切替

### 1.1 新規管理者を追加する

1. 対象者に LIFF 画面を開いてもらう
2. ブラウザの DevTools Console で `liff.getProfile()` を実行（または運営側がサポート）
3. 取得した `userId`（`U...`）を `ADMIN_LINE_USER_IDS` に追加
4. Vercel で再デプロイ（環境変数更新後の再ビルドは不要 — 次回関数実行時に反映）

---

## 2. カレンダー運用

### 2.1 日常チェック

1. 毎朝、管理画面の **カレンダー** タブを開き当日の予約を確認
2. 「予約●件」をタップして詳細パネルを開く
3. 予約キャンセル対応は詳細パネルの 🗑️ から（※取り消し不可）

### 2.2 手動で予約を入れる

- 「＋ 予約を追加」ボタン
- 利用者名 欄に displayName を入力すると既存ユーザーに紐付け、空欄なら管理者名義
- PIN は即時発行される（RemoteLock 連携済）

---

## 3. イベント・スクール運営

### 3.1 イベント作成

| 項目 | 入力例 |
|---|---|
| タイトル | GW スポーツイベント |
| 拠点 / 施設 | ASP / 体育館 |
| 開催日 | 2026-05-03 |
| 時間 | 10:00 – 12:00 |
| 定員 / 料金 | 20 / 1500 |

### 3.2 スクール作成（一般募集型）

- 曜日・開始〜終了時刻・総回数・料金（1 回あたり）を入力
- 利用者は **一括申込** で全回分を前払い（Stripe 決済）
- カレンダー上で毎週の開催日に緑バッジが自動展開

### 3.3 定期利用（団体・S-01 等の請求書払い）

> **オンライン決済は発生しません。** 月次の利用実績をもとに集計後、別途請求書を発行する運用です。

- **会員区分マスタ** で `discountType=FREE`（または相当）の会員区分を作成
- **会員管理** タブで対象ユーザー（団体代表）に会員区分を付与
- **管理者が代理予約**: **カレンダー → ＋ボタン**（または管理者用予約作成）から、対象ユーザー名で年度分を一括登録
  - 会員区分が `FREE` の場合、合計請求額は ¥0（PIN は通常通り発行）
  - 元金額（割引前）は `originalPrice`、割引額は `memberDiscount` として保存される
- **利用者側の挙動**:
  - 一括登録された予約は **マイ予約** に表示される
  - **キャンセルは利用開始 1 時間前まで可**（自分でキャンセル可能）
  - **追加予約は通常フローで可能**（FREE 会員区分が自動適用される）
  - **時刻・施設の変更は不可** → キャンセル後に取り直し、または運営に連絡（管理者が代理修正）
- **利用者の決済画面**: FREE 会員区分のユーザーには **「請求書払い（後日請求）」**カードのみが表示され、クレジットカード／PayPay は出ません。フッターの金額表示も「請求書払い」、ボタンは「予約を確定する」に切り替わります。

## 4. 会員・クーポン運用

### 4.1 会員種別の変更

- **会員区分マスタ** タブ → 既存種別の編集
- `discountType` を変えた場合、既存予約には遡らない（新規予約から適用）

### 4.2 会員付与

- **会員管理** タブ → ユーザー検索（displayName）→ 会員種別を選択 → 有効期間を設定
- `FREE` 区分は慎重に（無料で予約が成立する）

### 4.3 クーポン発行

1. **クーポン** タブ → ＋作成
2. `code` は半角英数の大文字推奨
3. `maxUses` で発行枚数を制限できる
4. 配布は LINE 公式アカウント・ポスター等で

---

## 5. お知らせ

- **お知らせ** タブから追加／編集
- `priority` は `info`/`warning`/`critical`
- `location` を指定すると拠点別に表示
- 利用者画面ではトップに常時表示される

---

## 6. 売上管理

### 6.1 集計対象

- **売上** タブ → 日別 / 月別 / 拠点別
- 集計対象は `status == PAID` の `checkins`（FREE 会員区分の予約も `PAID` で記録されるが `totalPrice=0` のため売上額には影響しない）

### 6.2 売上計上日（タイムラグの扱い）

- **売上計上日 = `date`（利用日）** を採用
- 例: 4/30(水) 20:00〜21:00 の予約 → **4 月の CSV に確実に含まれる**（与信日／確定処理日に依存しない）
- LaBOLA で発生していた「月末利用 → 翌月計上」のラグは **新システムでは発生しません**
- `createdAt` は予約作成日時として CSV にも別途記録（参考用）

### 6.3 詳細 CSV の項目

「詳細 CSV」ボタンで以下の列を出力します（Excel ピボットでの集計を想定）。

| 列 | 内容 |
|---|---|
| 利用区分 | `一般` / `定期`（会員区分が紐付いている予約は `定期`） |
| 会員区分(団体名) | `memberTypeName`（S-01 等の名称・団体名としても運用可） |
| 時間帯区分 | `平日昼` / `平日夜` / `土日祝`<br>※ 夜間は **開始時刻 18:00 以降**。**土日祝は曜日判定＋日本の祝日カレンダー（`@holiday-jp/holiday_jp`）** に連動。祝日は WEEKEND 料金が自動適用されます |
| 元金額(税込) | 割引適用前の金額 |
| 会員割引 | 会員区分による割引額（FREE の場合は元金額相当） |
| クーポン割引 | クーポン適用額 |
| 請求額(税込) | 実際の請求額 |
| 請求額(税抜) | `Math.floor(請求額(税込) ÷ 1.1)` |

### 6.4 一般 / 定期(S-01) の集計手順

詳細 CSV を Excel で開き、ピボットテーブルで以下のように集計してください。

**① 一般**
- フィルタ: `利用区分 = 一般`
- 行: `時間帯区分`（平日昼 / 平日夜 / 土日祝）
- 値: 件数、`請求額(税抜)` の合計

**② 定期（S-01 等）**
- フィルタ: `利用区分 = 定期`
- 行: `会員区分(団体名)`、`時間帯区分`
- 値: 件数、`元金額(税込)` の合計、`会員割引` の合計、`請求額(税抜)`<br>※ 定期の請求額は ¥0 のため、請求書ベースは元金額（または割引額）から起こす運用

## 7. 入館対応・現地トラブル

| 状況 | 対応 |
|---|---|
| PIN が通らない | 予約時間帯を再確認／RemoteLock コンソールでデバイス状態確認／ランダム PIN フォールバック疑い |
| 電池切れ | RemoteLock の電池残量通知を定期確認（台帳参照） |
| 誤予約（利用者から） | 利用開始 1 時間前までは利用者自身でキャンセル可／それ以降は管理画面でキャンセル |

## 8. 毎週／毎月の運用タスク

### 8.1 週次

- [ ] 予約状況の確認（カレンダー）
- [ ] レビュー閲覧（運営改善へのヒント）
- [ ] RemoteLock 電池残量のチェック

### 8.2 月次

- [ ] 売上 CSV を経理へ共有
- [ ] クーポン使用状況の確認（使い切り／停止判断）
- [ ] Firestore のエクスポート確認（[07_動作環境構築手順 §バックアップ](./07_動作環境構築手順.md#5-バックアップ--リストア)）

## 9. 監視・アラート

| 項目 | 監視方法 |
|---|---|
| 決済失敗 | Stripe ダッシュボードの通知メール |
| Webhook 失敗 | Stripe → Developers → Webhooks → Attempts |
| RemoteLock API 失敗 | Vercel ログで `RemoteLock API error` を検索 |
| Firestore 障害 | Firebase Console のアラート |
| 操作ログ | Vercel ログで `[OPERATION]` を検索 |

> 重大障害時は LINE 公式アカウントで告知＋お知らせ機能（`critical`）を投稿。

---

## 10. セキュリティ・個人情報

- LIFF 経由で取得するのは `userId`（内部 ID）／`displayName`／`pictureUrl` のみ
- カード情報は **Stripe** が保持し、本システムには保存しない
- 領収書 PDF には氏名・利用日時・金額が含まれる（データ受渡しに注意）
- バックアップ／ログは **アクセス制限されたストレージ** に保管
