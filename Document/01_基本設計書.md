# 01. 基本設計書

**システム名**: みんなの体育館 チェックインアプリ（`gym-checkin`）
**形態**: LINE ミニアプリ（LIFF）
**対応拠点**: みんなの体育館 ASP / みんなの体育館 やばせ
**最終更新**: 2026-04

---

## 1. 目的・背景

無人運営の体育館・トレーニングルームにおいて、利用者が LINE から予約・決済し、決済完了と同時に発行される 4 桁の PIN で入館できるセルフサービス型予約システムを提供する。

### 主要な課題と対応方針

| 課題 | 対応方針 |
|---|---|
| スタッフ不在での入退館管理 | RemoteLock スマートロックに API 経由で PIN を動的発行 |
| LINE ユーザーへの到達性 | LIFF（LINE ミニアプリ）として公式アカウントから起動 |
| 決済 | Stripe Checkout を利用（クレジットカード／PayPay 等） |
| 運営の業務効率 | 管理者用カレンダー／売上／イベント／スクール／クーポン管理を統合 |

## 2. システム全体構成

```
┌──────────────────────────────────────────────────────────────┐
│                    LINE ミニアプリ (LIFF)                      │
│  React 18 + TypeScript + Vite + Zustand + Tailwind CSS       │
│  React Router / TanStack Query                                │
└───────────────────────────┬──────────────────────────────────┘
                            │ HTTPS (Bearer: LIFF accessToken)
                            ▼
┌──────────────────────────────────────────────────────────────┐
│          Vercel Serverless Functions (/api)                  │
│  @vercel/node · Node 20.x                                    │
│  認可: LIFF access_token → LINE /v2/profile で検証           │
└──┬───────────────┬───────────────┬───────────────┬───────────┘
   │               │               │               │
   ▼               ▼               ▼               ▼
Firestore      Stripe          RemoteLock      LINE Login API
(Firebase)     Checkout        Booking API     Messaging API
                               (PIN 発行)
```

### 2.1 役割と責務

| コンポーネント | 役割 |
|---|---|
| LIFF フロント | 画面表示／ユーザー操作／LIFF 認証／LINE Profile 取得 |
| `/api` (Vercel) | ビジネスロジック／外部 API 連携／Firestore 操作 |
| Firestore | 予約・ユーザー・クーポン等の永続化 |
| Stripe | 決済・Webhook |
| RemoteLock | PIN 発行／施錠制御 |

### 2.2 ネットワーク・ドメイン

- 本番 URL: Vercel デプロイ URL（カスタムドメイン設定可）
- API: `https://<app>/api/*`
- LIFF URL: `https://liff.line.me/<LIFF_ID>`

## 4. 料金仕様

### 4.1 ASP

| 施設 | 平日昼 (08-17) | 平日夜 (17-21) | 土日祝 |
|---|---|---|---|
| 体育館 | ¥2,200/h | ¥2,750/h | ¥2,750/h |
| トレーニングルーム（貸切） | ¥2,200/h | ¥2,200/h | ¥2,200/h |
| トレーニングルーム（相席） | ¥550/人 | ¥550/人 | ¥550/人 |

### 4.2 やばせ

| 施設 | 平日昼 (07-17) | 平日夜 (17-21) | 土日祝 |
|---|---|---|---|
| 体育館 | ¥1,650/h | ¥2,200/h | ¥2,200/h |

### 4.3 営業時間

- **ASP**: 08:00 – 21:00
- **やばせ**: 07:00 – 21:00

### 4.4 定員

- 体育館 / 個室トレ: 同時間帯 1 組
- 相席トレ: 同時間帯 10 名

### 4.5 割引

| 種類 | 計算順序 |
|---|---|
| 会員種別割引 (`discountType` = `FREE`/`PERCENTAGE`/`FIXED_PER_HOUR`) | 1. 各日料金に対して適用 |
| クーポン (`discountType` = `PERCENTAGE`/`FIXED`) | 2. 会員割引後の合計に対して適用 |

最終金額 `= max(0, 通常料金合計 − 会員割引合計 − クーポン割引)`

### 4.6 祝日対応

- 日本の祝日は **`@holiday-jp/holiday_jp`** ライブラリで判定
- 祝日は土日と同様に **WEEKEND 料金** を適用
- 売上 CSV の `時間帯区分` も祝日を「土日祝」に分類

### 4.7 支払い方法

| 種別 | 表示 | 確定 |
|---|---|---|
| 一般利用 | クレジットカード／PayPay（Stripe Checkout） | 決済完了後に PIN 発行 |
| 定期利用（`discountType=FREE`） | **「請求書払い（後日請求）」のみ** | 予約確定と同時に PIN 発行（オンライン決済なし） |

## 5. 非機能要件

| 項目 | 要件 |
|---|---|
| 可用性 | Vercel / Firebase / Stripe / RemoteLock の SLA に準拠 |
| 性能 | Vercel Functions コールドスタート込みで 2 秒以内の応答を目標 |
| スケーラビリティ | Serverless・Firestore ともに自動スケール |
| セキュリティ | LIFF accessToken を LINE プロファイル API で検証／管理者は `ADMIN_LINE_USER_IDS` で制御 |
| 監査 | API/フロントとも操作ログを構造化出力（`logger.ts`） |
| 個人情報 | LINE の userId/displayName/pictureUrl のみを保持 |
| バックアップ | Firestore の自動バックアップ（日次エクスポート推奨） |
| 多言語 | 日本語のみ |
| ブラウザ | LINE 内 WebView（iOS/Android）を主対象 |

| サービス | プロトコル | 用途 |
|---|---|---|

## 8. 制約事項

| 制約 | 内容 | 対策 |
|---|---|---|
| Vercel Hobby プラン関数上限 | 12 個まで | 既存エンドポイントにクエリ `type`/`action` を持たせて集約 |
| Firestore 複合インデックス | 複数条件クエリ時に必要 | 主キー 1 つで取得→メモリ側で絞り込み＆ソート |
| RemoteLock API 障害 | PIN 発行不可 | ランダム PIN フォールバック（別途管理が必要） |
